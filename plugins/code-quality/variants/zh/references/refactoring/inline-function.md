# 内联函数（Inline Function）

## 什么是内联函数

内联函数（Inline Function）是 [extract-function.md](./extract-function.md) 的逆操作：它用函数体替换调用并删除该函数。提取函数（Extract Function）将一个概念抽取为一个名称，而内联函数则在名称不再值得保留时——当函数体表达的内容和调用一样多，或者表达得更清晰时——将函数折叠回去。

```python
# before
def get_rating(driver):
    return 2 if more_than_five_late_deliveries(driver) else 1

def more_than_five_late_deliveries(driver):
    return driver.late_deliveries > 5

# after
def get_rating(driver):
    return 2 if driver.late_deliveries > 5 else 1
```

## 何时内联

- **误导性的名称。** 当函数名比函数体对自身的描述更差时，间接会主动误导。内联移除了错误的指示牌。
- **琐碎的委托。** 一个除了转发什么都不做的函数——[thin-wrapper-function.md](./thin-wrapper-function.md) 中描述的过薄包装。当背后没有边界、没有测试接缝、没有不稳定的依赖时，内联它。
- **函数体比调用更清晰。** 有时一个一行函数隐藏了一个每个读者已经理解的简单表达式。调用迫使跳转去确认某个显而易见的事情；内联让读者的视线停留在一处。
- **撤销过早的提取。** 当一次较早的重构（通常是代理生成的）沿着错误的接缝将代码切碎成许多小的辅助函数时，将其中几个内联回一起可以让你看到真正的形状并沿着更好的边界重新提取。Fowler 的建议：当间接无用时，先内联，再重新提取。

## 内联前的安全检查

- **多态和重写。** 不要内联一个在子类中被重写的方法，或者一个其分发很重要的方法——内联会折叠类型系统在之间选择的行为。
- **递归和多个调用点。** 递归函数不能天真地内联。如果有许多调用方，到处内联可能是一个大的变更；在这样做之前考虑该函数是否实际上值得保留，并逐个调用点内联。
- **副作用和求值顺序。** 确保折叠函数体不会改变表达式求值的时机，特别是当参数有副作用或函数体读取可变共享状态时。
- **行为保持。** 与所有重构一样，每次内联后运行测试。参见 [safe-refactoring.md](./safe-refactoring.md)。IDE 提供"内联方法"，它处理机械性替换并更新所有调用点。

## 何时不内联

当函数在不稳定实现上提供稳定接口、作为测试接缝、命名了有助于搜索和理解的真正领域概念、或隐藏了真正的复杂性时，保留它。短小本身从来不是内联的理由——一个在清晰边界上的、命名良好的小函数是好的设计，而不是坏味。问题始终是名称和边界是否增加了含义，而不是它们后面有多少行代码。
