# 基本类型偏执（Primitive Obsession）

## 什么是基本类型偏执

基本类型偏执是一种习惯：用原始的基本类型——字符串、整数、浮点数、裸元组和未类型化的字典——来表示领域概念，而使用专门的类型本可以增加安全性和清晰度。用户 ID"只是一个字符串"，货币金额"只是一个浮点数"，订单状态"只是这些字符串值之一"，坐标"只是两个数字的元组"，解析后的记录"只是一个 `dict[str, Any]`"。每个决定单独看都很方便，但集体上它们将领域从代码中抹去了。

代价是基本类型不携带含义，也不执行任何规则。一个接受 `(str, str, str)` 的函数无法帮助调用方记住哪个字符串是邮箱、哪个是名字、哪个是城市——也无法阻止它们以错误的顺序传递三个值。用于金额的 `float` 会诱使用于数量的 `float` 以错误的单位与之相乘。用字符串表示的状态可以包含 `"shippped"`，直到生产环境才有人注意到。类型系统本可以捕获这些，却被告知了什么也不用做。

## 信号

- **验证散落在各调用点。** 同样的 `if not re.match(EMAIL_RE, value)` 出现在每个接收邮箱字符串的地方，因为字符串本身不保证任何东西。值对象在构造时验证一次。
- **以固定组形式出现的基本类型。** 纬度和经度总是一起传递，`currency` 和 `amount_cents` 总是并列——这些数据泥团（Data Clumps）想成为一种类型。
- **字符串类型的状态和种类。** 一个本应是枚举的字段却存为自由字符串，在代码中与字符串字面量比较。
- **`dict[str, Any]` 作为伪记录。** 已知固定形状的数据以字典形式传递，每次访问都是字符串类型的查找，类型检查器无法验证，拼写错误时在运行时失败。
- **类型耦合的行为无处安放。** 操作该基本类型的函数（格式化电话号码、标准化代码）堆积为自由函数，因为没有类型可以依附。

## 何时引入领域类型

当某个概念有自己的验证规则、不变量或关联行为时，引入一个**值对象**（在 Python 中是 `frozen` dataclass、`NamedTuple` 或小类）——金钱、邮箱、日期范围、坐标。当一个值是固定已知集合中的一个时，引入一个**枚举**（`Enum`、`StrEnum`）；这使拼写错误变成错误，并在 `match` 中提供穷尽性检查。当 `dict[str, Any]` 具有已知形状时引入一个**类型化记录**（`dataclass`、`TypedDict`、Pydantic 模型），以便字段访问被检查且结构由类型文档化。

回报是集中化：验证在类型构造的边界处发生一次，不变量在类型使用的任何地方都成立，行为有一个明显的归属。非法状态变得不可表示，而不仅仅是不太可能。

## 何时基本类型没问题

领域类型不是免费的——它们增加了定义、构造步骤和一层间接——过度应用也有自己的坏味。在以下情况下选择基本类型是正确的：

- **内部管道。** 循环索引、临时计数、局部累加器——这些确实只是数字，包装它们增加了噪声但没有安全性。
- **真正通用的代码。** 序列化器、缓存、通用容器或日志辅助函数应该对*任何*值都工作，不需要知道你的领域类型；基本类型和泛型在这里是正确的。
- **性能关键的路径。** 将大型数值数组的每个元素包装在值对象中可能是真实的成本。在热数值代码中，基本类型数组（或 NumPy dtypes）是合适的，领域含义在其他地方文档化。
- **该概念没有规则和行为。** 如果一个字符串只是一个从不验证、不与固定集合比较、也不被操作的透明标签，一个 `NewType` 别名可能就是它所需要的全部清晰度——或者什么都不需要。

判断在于该概念是否*赢得*了一个类型：包装它是否能防止一类错误、集中一个规则或给行为一个家？如果是，引入该类型。如果只是增加仪式感，保留基本类型。

## 在 Python 中

- `dataclass(frozen=True)` 是默认的值对象：不可变、可比较、定义成本低。
- `StrEnum` / `IntEnum` 用于还需要在边界（序列化、存储）上以基本类型表示的固定集合。
- `TypedDict` 用于必须保持字典形状（例如 API 边界的 JSON）但希望键被检查的情况。
- `NewType` 用于零开销的不同别名，当你想让类型检查器阻止 `UserId` 和 `OrderId` 互换，而不需要运行时包装。
- 在系统边界处（解析输入、读取数据库）构造领域类型，使得类型化的核心从不处理原始基本类型，类似于函数核 / 命令式外壳的划分。

基本类型偏执经常与 [feature-envy.md](./feature-envy.md)（嫉妒一个无法附加行为的基本类型）和数据泥团同时出现；引入缺失的类型常能同时解决多个坏味。
