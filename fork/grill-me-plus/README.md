# grill-me-plus

基于 Matt Pocock 的 `grilling` 与实验性 `batch-grill-me` 改造的结构化审视 Skill。它先从既有材料和环境中调研事实，维护追加式长期记录，再通过依赖感知的问题轮次审视计划、项目、决策或想法；每轮最多提出三个会实质改变结果且必须由用户决定的独立问题，轮数不设上限，并根据稳定的全局审视地图报告整体就绪度。

## 改造内容

- 从目标、完成标准、证据、范围、替代方案、假设、执行边界和风险等相关维度建立稳定的顶层审视地图，先检查全局覆盖范围，再深入具体分支。
- 默认维护文件操作级只追加的长期记录，持续保存调研、证据、分析、问题、讨论、决定、纠正、进度和剩余工作；没有指定位置时，先检查上下文，建议一个具体路径并征得同意，已有内容除修复意外损坏或用户明确要求外不得替换或删除。
- 在提出实质问题前进行与影响程度相称的最大努力调研，优先完整读取用户点名材料和项目现有证据；能由证据确定的事项直接形成结论，客观事实缺口不转嫁给用户猜测。
- 根据审视地图在内部生成候选问题，先以证据过滤，再按方向影响、错误代价、可逆性和下游解锁程度排序，只询问用户拥有的目标、偏好、取舍、授权、范围和风险决定；没有候选问题通过筛选时直接给出结论，不把必要产物、调研验证或常规下一步包装成用户选择。
- 存在至少三个独立高价值问题时默认提出三个；经过调研后不足三个、当前仍在深入讨论，或存在未决依赖时可以少问。三个始终是单轮上限，不能为了凑数降低标准。
- 每个编号承载一个连贯的决定主题，问题编号在整个审视中全局递增，换轮、checkpoint、上下文压缩、会话恢复或 Skill 重载时不重置。
- 默认给出事实或判断、建议做法及理由，再开放式提问。只有存在多个真实可行且取舍不同的做法时才提供“首选 + 其他做法”，不捏造选项，也不要求编码式作答格式。
- 集中确认低风险、可撤销的默认项，但不把高影响决定隐藏在默认项中。
- 用户深入讨论、质疑建议或提出疑问时暂停问题队列，先解决当前讨论，再重算审视框架与当前可推进事项并恢复访谈。
- 根据稳定的顶层审视地图和完成标准报告一个粗略的定性全局百分比，并说明推动和限制该判断的关键因素；百分比不通过问题、材料、决定或其他动态事项计数计算，旧的叶子计数快照不能沿用为当前进度，设计与行动就绪度只做定性分轴说明。
- 在初始地图、约每五轮、主要分支或范围变化、用户询问及结束前提供 checkpoint；阶段总结不是停止条件。
- 仅在相关审视维度已经覆盖、没有会改变结论或下一步的未决实质事项时收束，并报告提前结束时的剩余阻塞项。
- 保留最终确认前不实施被审视主题的边界，并明确调研和维护审视记录不属于提前实施；允许用户明确要求边决策边行动。

## 语言变体

```text
fork/grill-me-plus/
├── en/
│   └── grill-me-plus/
├── zh/
│   └── grill-me-plus/
├── README.md
└── upstream.toml
```

- `en/grill-me-plus/` 是英文源版本，也是非中文 locale 下的默认安装变体。
- `zh/grill-me-plus/` 是由英文版本翻译的中文审计版本，保持相同的结构和行为。
- 需要明确选择中文版本时，使用 `uv run scripts/install.py install grill-me-plus --variant zh`。

本 Skill 禁止隐式调用，避免普通方案讨论意外进入持续追问模式。
