# Duplicated Code

## 它是什么

Duplicated Code 指的是同一件事在多个地方被表达出来。这种 smell 被归进 dispensables，是因为真正消除重复通常会让代码更清晰、修改成本更低。但它也是最容易被误读的 smell，因为表面症状 - 两段看起来相似的片段 - 并不能说明是否真的有问题。整个判断都取决于一个区别。

## 知识重复 vs 偶然相似

这个区别最关键，而且判断错了，往任一方向都会出问题。

**知识重复** 是同一个 _决策_ 在多个地方被表达出来：一条业务规则、一个验证条件、一个 schema、一条协议细节、一次税费计算、一个 state transition。判断方法是反事实式的：_如果其中一份副本背后的需求变了，其他副本也必须跟着变吗？_ 如果答案是“是”，说明它们编码的是同一条知识，分散在多个地方意味着未来修改时会只改一部分、漏掉另一部分，导致系统自相矛盾。这种重复值得去掉。

**偶然相似** 则是今天看起来像、但原因不同、驱动力也不同的代码。两个结构相似但检查不同概念的 validator；两个有平行样板、却逻辑无关的 handler；长得像别的 test setup、但锁定不同场景的测试准备。如果其中一份改了，其他并不会动。这些片段并不是真正意义上的重复 - 它们只是“押韵” - 把它们统一起来只会把本不该耦合的东西绑在一起，造出错误的抽象。这里所依赖的原则见 [duplicated-code.md](./duplicated-code.md) 的兄弟文档 [../design-principles/dry.md](../design-principles/dry.md)：DRY 讲的是知识，不是文本。

## 何时提取

当你确认这些片段编码的是同一条知识时，就可以提取。对于 **高风险知识** - 安全、权限、金钱、协议契约、数据一致性、schema 的唯一事实来源 - 即使在第二次出现时也应该立刻提取。因为这些地方一旦分叉，代价太高，不能等。

对于普通代码，如果你大致确定它们相关，但变化方向还不清楚，**三次规则** 就是刹车：写一次，容忍两次；到第三次 - 或者变化方向已经很明确时 - 再提取。到第三次时，你通常已经能看出真正变化的是什么、不变的是什么，这样就能选对边界，而不是瞎猜。见 [../design-principles/rule-of-three.md](../design-principles/rule-of-three.md)。

## 何时保留

当片段只是偶然相似，或者它们确实有关联但还没有清楚到足以提取、而且你还没到第三次时，就保留重复。过早统一会选错抽象，而错误抽象比它替代的重复更贵 - 你先要付钱把它建出来，之后还要再付钱把它拆掉，因为场景会分化。暂时保留两份副本成本低、也容易回退；不想要的抽象却两者都不是。

## 抽象稳定性要求

提取之前，抽象的形状必须足够稳定。如果你还无法给要提取的东西起一个合适的名字，或者你已经能预见下一种情况会需要不同的参数或分支，那这个边界还没准备好。一个好的提取应该名字清楚、参数少、只有一个修改理由，并且能隐藏真实复杂度。如果候选项做不到这些，那重复是在告诉你：底层概念还没稳定 - 先等等。

## 错误去重

最常见的失败，尤其在 agent-assisted code 里，是 **参数化 wrapper 比重复本身更难读**。你把两段相似代码提成一个函数，但它们并不完全相同，所以这个函数开始需要一个 boolean flag 来切换行为，接着是一个 mode 参数，再接着是一个 callback 来处理真正不同的部分，然后为边缘情况又多出一个特殊分支。结果是一个没人能读的函数，被一堆调用点配置得乱七八糟 - 比两份老老实实的副本更糟。遇到这种情况，修法通常是把它 inline 回去，把代码重新复制开，再去找真正的变化轴。一个用 flag parameter 在两种行为之间切换的函数，往往说明它本该保留成两个函数。

## 在 Python 中

- 对重复的 schema、API contract 或 model，建立单一事实来源：`dataclass`、`TypedDict`、Pydantic、OpenAPI spec 或代码生成 - 不要手工维护三份副本。
- 表驱动映射和 dispatch dict 能把真正重复的知识（每个 case 一行）折叠起来，而不必硬造 class hierarchy。
- 优先提取成模块级函数，或者小型 strategy function，而不是做一个只为了共享代码的 base class。
- 留意同一条领域规则同时出现在 schema、service 和 test fixture 中 - 这是 agentic coding 很常见的模式：三份“副本”会悄悄分叉。

具体转换通常是 [extract-function.md](./extract-function.md)；当你发现自己做了错误去重，反向修正通常是 [inline-function.md](./inline-function.md)。
