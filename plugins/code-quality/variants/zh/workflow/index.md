# 工作流（Workflow）

本技能以三种模式之一运行。阅读对应文档，然后执行。

| 模式 | 适用时机 | 文档 |
|-|-|-|
| 快速审查（Fast Review） | 默认模式。日常开发后自查、小型 diff、单文件或 PR 审查。 | [fast-review.md](./fast-review.md) |
| 完整审查（Full Review） | 仅当用户明确要求"完整审查"、"架构审查"、"系统性审查"或"重构评估"时。 | [full-review.md](./full-review.md) |
| 分析（Analysis） | 设计讨论、模式/范式比较、重构规划、结构设计。无需评估 diff。 | [analysis.md](./analysis.md) |

默认使用快速审查。不要自行升级到完整审查——它更重，必须由用户触发。当用户询问*该做什么*而非*哪里有问题*时，切换到分析模式。

三种模式默认均为只读。仅在用户要求修复时修改代码，并在任何不安全、批量或跨文件的变更前停下来等待确认。
